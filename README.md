# letter-can

送信先を指定するための、CAN上のプロトコル。

## 前提

CAN Classicの拡張フォーマットデータフレーム[^1]のIDを使用します。

[^1]: [はじめてのCAN/CANFD p20参照](https://cdn.vector.com/cms/content/know-how/VJ/PDF/For_Beginners_CAN_CANFD.pdf)

## ノード識別子

letter-can上の各ノードは8bitの識別子を持つ必要があります。

## プロトコル

### 通常フォーマット

```
|匿名フラグ 1bit|送信先識別子 8bit|送信元識別子 8bit|未使用領域 12bit|
```

送信に対する応答が必要な場合に使用します。pingや、フィードバックのリクエストに使用します。

送信元が特定可能で、匿名フラグは無効(0)です。

### 匿名フォーマット

```
|匿名フラグ 1bit|送信先識別子 8bit|未使用領域 20bit|
```

送信に対する応答が必要ない場合に使用します。

送信元情報を含まず、匿名フラグが有効(1)です。
